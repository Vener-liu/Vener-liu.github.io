mariadb配置文件失踪

2017-12-01 Linux杂货 Linux 评论

使用源码安装包安装mariadb的时候，配置应该是没有错的，然而安装完发现/etc目录下并没有.cnf
配置文件。
于是
[root@server1 ~]# vim /etc/my.cnf
[client]
port = 3306
socket = /tmp/mysql.sock
default-character-set = utf8mb4

[mysqld]

然后登录数据库
mysql(none)>show global variables \G
浏览了一下配置，惊奇的发现port的值为0
然后怀疑配置文件除了问题

排错步骤：
1.在/etc/my.cnf 文件中改变其中一个配置的值
systemctl restart mysqld
然后登录mysql发现该配置的值也随之改变

2.命令行中（需要把mysql的bin目录加进环境变量）
mysql –help|grep ‘my.cnf’
/etc/my.cnf /etc/mysql/my.cnf ~/.my.cnf
order of preference, my.cnf, $MYSQL_TCP_PORT,
表示配置文件的加载顺序，越前面越高的优先级.这里表示/etc/my.cnf配置文件会优先被加载

3.ps -ef|grep mysqld
root 10971 1 0 12:22 ? 00:00:00 /bin/sh /usr/local/mysql/bin/mysqld_safe –datadir=/data/mysql –pid-file=/data/mysql/mysql.pid
mysql 11242 10971 0 12:22 ? 00:00:00 /usr/local/mysql/bin/mysqld –basedir=/usr/local/mysql –datadir=/data/mysql –plugin-dir=/usr/local/mysql/lib/plugin –user=mysql –log-error=/data/mysql/mysql-error.log –open-files-limit=65535 –pid-file=/data/mysql/mysql.pid –socket=/tmp/mysql.sock –port=3306
root 11963 7842 0 12:28 pts/1 00:00:00 grep –color=auto mysqld

我们可以看到mysqld进程是存在的，而且配置文件中写的也是3306端口

以上3个步骤中说明配置文件被正确加载，也就是说，有可能是配置中某一项的配置除了问题，网上查找了资料
把skip-networking这行注释掉，端口的值就加载出来了。
启动，并进入mysql后，输入命令：show global variables like ‘port’;
