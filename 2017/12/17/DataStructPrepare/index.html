理论知识准备

  • 数据结构（学科角度）：是一门研究非数值计算的程序设计问题中的操作对象，以及他们之间的关系和操作等相关问题的学科
  • 数据：是描述客观事物的符号，是计算机中可以操作的对象，能被计算机识别，并输入给计算机处理的符号集合
  • 数据元素：是组成数据的，具有一定意义的基本单位，在计算机中通常作为整体来操作。也称为记录
  • 数据项：一个数据元素由多个数据项组成，是数据中不可分割的最小单位
  • 数据对象：是性质相同的数据元素的集合，是数据的子集

  • 逻辑结构：是数据对象中的数据元素之间的相互关系,有四种：
    1.集合结构：数据对象中的数据元素除了同属于一个集合外，它们之间没有其他关系
    2.线性结构：数据对象中的数据元素存在一种一对一的关系
    3.树形结构：数据对象中的数据元素存在一种一对多的关系
    4.图形结构：数据对象中的数据元素存在一种多对多的关系

  • 物理结构：是指数据的逻辑结构在计算机上的存储形式
    1.顺序存储结构：是把数据元素存放在地址连续的存储单元里。它们的逻辑关系和物理关系是一致的
    2.链式存储结构：是把数据元素存放在任意的存储单元里，这些存储单元可以是连续的，也可以是不连续的

  • 数据类型：是一组相同性质的值的集合及定义在此集合上的一些操作的总称
    在C语言中分为两类：
    1.原子类型：是最基本类型单位，不可在分解，如int型，float型等
    2.结构关系：由多个类型组成，可以再分解。如int型数组就是有若干个int型组成

  • 抽象数据类型：是一个数学模型及定义在该模型上的一组操作（把一组操作抽象成数据类型）

  • 数据结构学科中的数据结构：是相互之间存在一种或多种特定关系的数据元素的集合
  • 算法的定义：是解决特定问题求解步骤的描述，在计算机中为指令的有限序列，并且每条指令都表示一个或者多个操作
  • 特性：0个或者多个的输入，1个或者多个的输出，确定性，可行性，有穷性
  • 要求：正确性，可读性，健壮性，时间效率高和存储量低
  • 算法效率的度量方法：
    1.事后统计法
    2.事前分析估算法（推荐）
  • 算法空间复杂度：表示算法的空间需需求
  • 算法时间复杂度：表示算法的时间需求
  • 推导大O阶方法：
    1.用常数1替代运算时间中的所有加法常数
    2.修改后的运行次数函数，只保留最高阶
    3.最高阶存在并且不为1，除去最高阶常数项
  • 时间复杂度排序：O(1)<O(㏒n)<O(n)<O(n㏒n)<O(n^2)<O(n^3)<O(2^n)<O(n!)<O(n^n)

共享空间顺序栈:

#define MAX_SIZE 10
typedef int ElemType
typedef struct {
    ElemType data[MAX_SIZE];

    ElemTupe top2;　//当 top1+1=top2 或　top2-1=top1　的时候栈为满

队列:是只允许在一段进行插入操作，而在另一端进行删除操作的线性表 FIFO
顺序队列:类似数组，一定要以0为队头，缺点较多。
循环队列:可以不把　0　当做是队头,循环移动队头队尾动态改变.　队尾指向的空间不存放任何数据
队满条件:队尾指针后移一位对队长取余等于队头指针　数学表达式：(queue->rear+1)%QUEUE_MAXSIZE
队空条件:queue->rear=queue->fornt
队长度：(queue->rear-queue->front+QUEUE_MAXSIZE) % QUEUE_MAXSIZE 就是队尾下标减去队头下标加上总长度然后对总长度取余
定义:

串:串是由零个或多个字符组成人那个的有限序列，又名叫字符串。字符串查找匹配的时候会用到
串的顺序存储结构：是用一组地址连续的存储单元来存储串中的字符序列。
串的链式存储结构：与线性表的链式存储结构类似，但结构中的结点可以存放一个过或者多个字符，比顺序存储灵活但性能较差

朴素的模式匹配算法：对主串S的每一个字符作为子串的开头，与要匹配的字符串T进行匹配。对主串S做大循环，每个字符开头做T的长度的小循环,直到匹配成功过着全部遍历完成为止。
朴素的模式匹配算法　-> KMP模式匹配算法 -> 改进KMP模式匹配算法 Boyer-Moore算法

树是n(n>=0)结点的有限集合，等于0时为空树
结点分为根结点、内部结点和叶结点
树中结点的最大层次称为树的深度或高度

二叉树：由一个根结点和左子树和右子树组成
度是是指结点拥有的子树数量
每个结点最多有两棵子树，左子树和右子树
特殊二叉树
斜树:所有结点只有左子树称为左斜树，所有结点只有右子树称为右斜树
满二叉树:所有叶结点在同一层次
完全二叉树:按顺序编号结点，从左到右开始连接结点，叶子结点只能出现在最下面两层;倒数第二层，若有叶子结点一定在右部连续;同样结点的二叉树，完全二叉树的深度最小
二叉树性质:
1.在二叉树的第i层上最多有２^(n-1)个结点 i>=1
2.深度为k的二叉树总共至多有２^k - 1 个结点　k>=1
3.对于任意一棵二叉树Ｔ其中段结点树为n0,度为2的结点树为n2，则n0=n2+1
4.具有n个结点的完全二叉树的深度为log2n +1对应第二个性质
5.如果一棵有n个结点的完全二叉树的结点按层序编号(从第1层到log2n+1层),对于任意一结点i:
①如果i=1;则结点i是二叉树的根,无双亲;如果i>1,则其双亲是编号为i/2的结点
②如果 2 i>n,则结点i无左孩子;否则其左孩子是编号为 2 i 的结点
③如果 2 i +1 >n,则结点i无右孩子;否则其右孩子是编号为 2 i + 1 的结点

二叉树的遍历:
前序遍历、中序遍历、后序遍历、层序遍历。循环遍历结点的方式
前序遍历:若二叉树为空,则空操作返回,否则先访问根结点，然后前序遍历左子树，再前序遍历右子树.
中序遍历:若二叉树为空,则空操作返回,否则从根结点开始,中序遍历根结点的左子树,然后是访问根结点,最后中序遍历右子树.
后序遍历:若二叉树为空,则空操作返回,否则从左到右先叶子后结点的方式遍历访问左右子树,最后访问根结点.
层序遍历:若二叉树为空,则空操作返回,否则从书的第一层也就是根结点开始访问,从上而下,从左到右逐层遍历.
