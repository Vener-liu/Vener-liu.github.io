当前环境：linux fedora 23 APACHE2.4 PHP5.6 mariadb10.0.28
Lamp环境搭建

ThinkPHP3.2.3完整版下载地址

下载完后把解压缩目录下面的 ThinkPHP 文件夹改名为放到网站根目录

thinkphp 3.2.3 的安装

  • 框架目录： 
    ├─index.php       入口文件 
├─README.md       README文件
├─Application     应用目录 
├─Public          资源文件目录 
└─ThinkPHP        框架目录框架系统目录（可以部署在非web目录下面） 
│  ├─Common       核心公共函数目录 
│  ├─Conf         核心配置目录  
│  ├─Lang         核心语言包目录
│  ├─Library      框架类库目录
│  │  ├─Think     核心Think类库包目录 （这个目录下有MVC对应类的父类）
│  │  ├─Org       Org类库包目录
│  │  ├─Vendor    第三方类库目录
│  │  ├─ ...      更多类库目录 
│  ├─Mode         框架应用模式目录
│  ├─Tpl          系统模板目录
│  ├─LICENSE.txt  框架授权协议文件
│  ├─logo.png     框架LOGO文件
│  ├─README.txt   框架README文件
│  └─ThinkPHP.php    框架入口文件
    #上述应用的目录结构只是默认设置，事实上，在实际部署应用的时候，我们建议除了应用入口文
    #件和Public资源目录外，其他文件都放到非WEB目录下面，具有更好的安全性。

第一站：虚拟站点

  • 遭遇：

         建设虚拟站点，到网上搜索教程，发现都是千篇一律：
     一、找httpd.conf 把# Include conf/extra/httpd-vhosts.conf 的注释去掉
     二、开始配置虚拟主机 http-vhost.conf中配置
     三、dns指向
     本环境中的apache版本是2.4.16
     发现并没有一中所指的 Include 配置，只好自己加，然后在/var/httpd/conf/ 中
     添加http-vhost.conf文件（原来并没有） 然后systemctl restart httpd 出现错误
     systemctl status httpd.service 中查看在http-vhost.conf中出现语法错误
     解决办法：
     一、首先进行DNS绑定
     在127.0.0.1 localhost 下面添加：
     127.0.0.1 shop.dev  
     127.0.0.1 www.shop.dev
     127.0.0.1 op.dev
     浏览了一下apache的配置文件httpd.conf 发现最后一行已经包含了所有配置文件的路径在conf.d下面
     所有并不用自己添加包含路径
     新建文件内容如下：

               <VirtualHost *:808>
              ServerAdmin webmaster@dummy-host2.example.com
              DocumentRoot "/var/www/html/shop"
              ServerName shop.dev
              ErrorLog "logs/dummy-host2.example.com-error.log"
              CustomLog "logs/dummy-host2.example.com-access.log" common
              <Directory "/var/www/html/shop">
                  #用于显示设定“可显示文件列表”(当无可显示网页的时候)
                  Options Indexes
                  #启用文件夹访问控制的文件.htaccess设置
                  AllowOverride All
                  #请求控制
                  Require all granted
                  #默认打开的页面设置
                  DirectoryIndex index.php index.html
              </Directory>
          </VirtualHost>

          <VirtualHost *:80>
              ServerAdmin mylocal
              DocumentRoot "/var/www/html/"
              ServerName shop.dev
              ErrorLog "logs/mylocal.com-error.log"
              CustomLog "logs/mylocal.com-access.log" common
              <Directory "/var/www/html">
                  #用于显示设定“可显示文件列表”(当无可显示网页的时候)
                  Options Indexes
                  #启用文件夹访问控制的文件.htaccess设置
                  AllowOverride All
                  #请求控制
                  Require all granted
                  #默认打开的页面设置
                  DirectoryIndex index.php index.html
              </Directory>
          </VirtualHost>

          <VirtualHost *:80>
              ServerAdmin op123
              DocumentRoot "/var/www/html/op123"
              ServerName op.dev
              ErrorLog "logs/op.com-error.log"
              CustomLog "logs/op.com-access.log" common
              <Directory "/var/www/html">
                  #用于显示设定“可显示文件列表”(当无可显示网页的时候)
                  Options Indexes
                  #启用文件夹访问控制的文件.htaccess设置
                  AllowOverride All
                  #请求控制
                  Require all granted
                  #默认打开的页面设置
                  DirectoryIndex index.php index.html
              </Directory>
          </VirtualHost>
          ServerAdmin 可以写自己邮箱
          DocumentRoot 项目/站点绝对路径
          ServerName DNS所指向的域名(这里要保持一致)
          ErrorLog 以及 CustomLog 都是用户日志文件路径，在/etc/httpd/logs 文件夹下
          <Directory>内容如注释所示
以上展示了以多端口设置项目站点方法和多域名设置项目站点方法实现虚拟站点功能，还有IP虚拟站点指向等

          值得注意的是：当使用相同IP、相同端口、不同域名的配置的时候，默认访问配置顺序加载的第一位
      以上面为例：当访问localhost的时候，会指向ServerName为shop.dev的站点。

第二站：目录权限

LAMP环境搭建使用dnf高级软件包管理工具分别单个安装

安装目录(主要配置文件) 

          apache 配置文件 /etc/httpd/conf/httpd.conf

           php  配置文件 /etc/php.ini

           mariadb(mysql)   /etc/my.cnf
                            /etc/mycnf.d/

  配置项目目录 （在网站根目录下新建一个shop目录）
    项目文件位于/var/www/html/shop 
                |---index.php =>                <?php
                                                define('APP_PATH','./app/');
                                                // 定义应用目录
                                                define('RUNTIME_PATH','./Runtime/');
                                                // 开启调试模式
                                                define('APP_DEBUG',True);
                                                include('../ThinkPHP/ThinkPHP.php');                                                  
                                                ?>

我把thinkphp下的核心文件Thinkphp并放在网站下 :/var/www/html/目录下

  • 遭遇： 

        在这之前，我在index.php 里面并没有加上三行define代码，查阅了手册之
    后才知道要加（上面是正常文见）

    导致 http://shop.dev/shop 访问 出现 系统错误 提示

    然后，再次执行，发现 ./shop目录不可写

    应该是出现权限问题

  • 解决办法： 

        `cat /etc/httpd/conf/httpd.conf`   查看apache的用户名和组名都是：apache  

    而我当前用户名是：carlos  而网站目录下面的文件都是属于 carlos:carlos 及都是属于
    carlos用户组的carlos成员
    管理员身份下： 
    `usermod -g carlos -G apache`;//将carlos成员加入到apache组，因为平常开发成员是
    carlos有可能会使用到这个apache组
   `cd /var/www/`

   `chown -R carlos:apache` ./html //把网站下面的所有文件都交属carlos:apache 即carlos用户组的apache成员 

   `chmod - R 775 ./html`  //设置目录访问权限 rwxrwxr_x 如果目录下面有特殊文件需要不同权限，可进入目录再更改权限

   `vim /etc/httpd/conf/httpd.conf`  //修改  User apache 为 User carlos

    `systemctl restart httpd`   //重启httpd服务

    文件权限修改完毕，接着关闭SELinux（一种访问控制体系）

   `getenforce`  #如果显示Disabled 就可以直接访问本地网址,否则继续

   `getenforce 0` #临时生效，重启失效

   `vim /etc/sysconfig/selinux` #重启生效



       完成入口文件引入！可以看看你项目的目录下面会有什么文件       <br>

第三站：apache模块开启

  • 我今天刚好要使用thinkphp的url重写

    在网上找教程，发现基本上都是要在/etc/httpd/conf/httpd.conf 中开启rewrite模块

    然而，从第一行找到最后一行也没有，根据第一个坑，我在httpd目录下面发现了个modules文件夹

    ls |grep rewrite

    mod_rewrite.so

    也就是说有这个模块，但是要找到地方开启。

    httpd目录下，有一个conf.models.d文件夹，打开里面几个文件后分析发现，这里应该是apache model模块的开启的地方，尝试打开几个文件后，在00-base.conf发现了这行

    LoadModule rewrite_module modules/mod_rewrite.so

    所以默认是启用rewrite模块的状态

mod_rewrite是Apache的一个模块。
此模块提供了一个基于正则表达式分析器的重写引擎来实时重写URL请求。
它支持每个完整规则可以拥有不限数量的子规则以及附加条件规则的灵活而且强大的URL操作机制。
此URL操作可以依赖于各种测试，比如服务器变量、环境变量、HTTP头、时间标记，甚至各种格式的用于匹配URL组成部分的查找数据库。
此模块可以操作URL的所有部分(包括路径信息部分)，在服务器级的(httpd.conf)和目录级的(.htaccess)配置都有效，还可以生成最终请求字符串。
此重写操作的结果可以是内部子处理，也可以是外部请求的转向，甚至还可以是内部代理处理。

一般mod_rewrite默认是不启动的（新版Apache 默认启动），这时我们需要手动启用它。

  • 解决方法：

    1、启用mod_rewrite模块
      打开Apache名为“httpd.conf”的配置文件，`#LoadModule rewrite_module modules/mod_rewrite. so`这行代码，将#注释符删除，使得Apache在启动时就加载mod_rewrite模块。<br>
    2、设置Apache
        这一步需要特别注意，在httpd.conf下的配置是根目录的配置，而我们的目录是虚拟站点，第一个坑里面提到
    我创建了一个站点shop而站点配置文件在/etc/httpd/conf.d/ 目录下面有一个虚拟站点配置文件
    http-vhost.conf
    找到项目虚拟站点的配置那一段
      将其中的：AllowOverride None 改为 AllowOverride All ，
      并且把 Options Indexes 注释掉
      添加 Options FollowSymLinks
      重启httpd服务即可。
  最终配置（项目路径要对得上）：

            <Directory "/var/www/html/shop">
          #启用允许访问符号链接文件
          Options FollowSymLinks
          #用于显示设定“可显示文件列表”(当无可显示网页的时候)
          # Options Indexes 
          #启用文件夹访问控制的文件.htaccess设置
          AllowOverride All
          #请求控制
          Require all granted
          Order allow,deny
          Allow from all
          #默认打开的页面设置
          DirectoryIndex index.php index.html
        </Directory>
  • 3、添加thinkphp的配置
          当前项目根目录下的 Common/Conf/config.php : 

              return array(
              //'配置项'=>'配置值'
              'URL_MODEL' => 2
          );
  • 4、在入口文件index.php同级目录添加.htaccess文件，内容如下：
          即当前项目根目录下创建.htaccess文件：
      <IfModule mod_rewrite.c>
      Options +FollowSymlinks
      RewriteEngine On
      RewriteCond %{REQUEST_FILENAME} !-d
      RewriteCond %{REQUEST_FILENAME} !-f
      RewriteRule ^(.*)$ index.php/$1 [QSA,PT,L]
      </IfModule>

好了OK！
http://域名/项目目录/分组/控制器/方法
就可以访问了，url中没有了index.php 文件
