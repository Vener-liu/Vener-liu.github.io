Hexo博客部署在Github

环境：linux(fedora23)

安装node.js （本机好像安装了比较低的版本，后面直接升级了）

安装

点击进入node.js官网
目前node.js有两个推荐版本，分为通用版和最新版，点击可直接进行下载。下载好后，按照既定的套路安装即可。
安装Node.js有好多种方法，一般从官网下载好安装包，点击安装即可（跟所有的Windows应用一样方便）。
安装好之后就可以使用了，可以在终端内测试Node代码。

升级

Node.js一直保持着很高的更新频率，为了体验它的最新功能，我们免不了要经常更新之，不能每次都下源码，再编译安装；或者是下载好编译好的文件设置路径吧。太麻烦了。。。
node提供了一个更新模块——n（它是专门用来管理Node.js版本的。
首先安装该模块（当然是用node好基友npm了）

npm install -g n

安装完成后，检测是否安装成功

n -V

安装成功会输出当前安装的n的版本号。

使用n更新node

n stable

安装Git

dnf install git

安装完成后：

git --version

显示当前git版本

Github仓库

github申请不啰嗦
在你的github仓库要创建一个xxx.github.io 仓库，本地部署完之后同步到这个仓库
注意：’xxx’ 就是你的github用户名！下面的创建的文件也是同样的名字

生成公钥（github）

$ssh-keygen

首先 ssh-keygen 会确认密钥的存储位置（默认是 .ssh/id_rsa），然后它会要求你输入两次密钥口令。如果你不想在使用密钥时输入口令，将其留空即可。

现在，进行了上述操作的用户需要将各自的公钥发送给任意一个 Git 服务器管理员（假设服务器正在使用基于公钥的 SSH 验证设置）。 他们所要做的就是复制各自的 .pub 文件内容，并将其通过邮件发送。 公钥看起来是这样的：

然后到github设置加入这个公钥就可以了！

本地创建博客文件夹

使用普通用户在其 ~(当前用户家目录)创建文件夹，不然会出现权限问题

例如：

cd ~
mkdir Yisen.github.io
cd Yisen.github.io
npm install -g hexo-cli
hexo init ./ #初始化到当前目录

npm install //  node.js的命令，根据博客既定的dependencies配置安装所有的依赖包,少了这一步后面有一个大坑等着你，卡死在hexo deploy 1h....的结论（不行再加上组合拳：npm install hexo-deployer-git --save）

初始化完成

首先安装个VIM

dnf -y update

dnf -y install vim-enhanced

选取主题：hexo主题

点进去相关项目的github仓库，有说明如何使用，包括配置（70%是英文 chrome可翻译）
一般要把下载主题包（已解压）放在博客根目录下的themes文件夹下

基本配置

在博客根目录

vim _config.yml

# Site
title: Yisen

subtitle: nice

description:

author: Yisen LIU

language: zh-CN

timezone: Asia/Shanghai

主题配置：

theme: pure //默认是landscape

拉到最后面部署配置：

新版hexo配置如下

deploy:
  type: git
  repo: http://github.com/Vener-liu/Yisen.github.io.git
  branch: master
  message: BLOG

也可以参考官网：hexo官方文档

域名配置：
在博客根目录新建一个文件：

echo '你的域名'>CNAME

vim _config.yml

找到：url: http://你的域名 进行修改

发表一篇文章
在终端输入：

hexo new "文章标题" // 新建一篇文章

//每次部署上github之前可以现在本地预览：
hexo server

部署步骤

每次部署的步骤，可按以下三步来进行。

    hexo clean

    hexo generate （hexo g）

    hexo deploy   (hexo d)

一些常用命令

hexo new"postName" #新建文章

hexo new page"pageName" #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本

删除文章步骤：

hexo clean

然后到source下的_post文件夹删除源文件

hexo clean

hexo g

hexo d

OK!

增加域名

向你的 Github Pages 仓库添加一个CNAME(一定要大写)文件

CNAME文件要放在source目录下

其中只能包含一个顶级域名，像这样：

yisenliu.top`` 或者二级域名也行 ``veners.oicp.io



在DNS做一下解析

A记录 (ip: ping一下github给你的域名就知道了)

CNAME (github给你的域名)

部署完毕
