Linux下使用Samba

2017-11-26 Linux运维基础 Linux 评论

一 安装Samba服务

首先，安装Samba以及进行一些基本的配置。
检验Samba是否已经安装在您的系统中：

rpm -q samba samba-common samba-client

如果上面的命令没有任何输出，这意味着Samba并未安装。这时，应使用下面的命令来安装Samba。

dnf install samba samba-common samba-client

二 设置共享文件夹

接下来，创建一个用于在网络中共享的本地文件夹。这个文件夹应该以Samba共享的方式导出到远程的用户。我在当前家目录中创建这个文件夹。
cd ~
mkdir shared

在创建/shared文件夹后，设置文件夹权限以保证其余用户可以访问它。

$ sudo chmod o+rw ～/shared
如果您不想其他用户对该文件夹拥有写权限，您需要移除命令中的’w’选项。

$ sudo chmod o+r /shared

三 启动samba服务

使samba服务器开机启动

systemctl enable smb.service

启动samba服务器

systemctl start smb.service

把关键字一查，发现需要更新依赖库

5、查看samba的服务启动情况

service smb status

testparm //查看配置状态

vim /etc/samba/smb.conf

四 配置文件

（在我的另一篇文章中有介绍详细配置信息）

然后是我的配置：

[global]
workgroup = MYGROUP
        server string = Samba Server Version %v

        netbios name = Share

        interfaces = lo eth0 wlp6s0 192.168.12.2/24 192.168.13.2/24
        hosts allow = 127. 192.168.43. 192.168.13.



[homes]
        comment = Home Directories
        browseable = no
        writable = no
        valid users = carlos


[myshare]

comment=my shared files

path=/home/carlos/share

;public=yes

writeable=yes

browseable = yes

available = yes

admin users=carlos

上面跟的admin users=carlos 是samba的用户，要先创建

cd ~

pdbedit –a carlos //输入密码两次就创建成功了

五 开放端口

防火墙需要开放端口：

firewall-cmd --permanent --zone=public --add-port=137/udp
firewall-cmd --permanent --zone=public --add-port=138/udp
firewall-cmd --permanent --zone=public --add-port=139/tcp
firewall-cmd --permanent --zone=public --add-port=445/tcp
或者
firewall-cmd --permanent --add-port=137-138/udp
firewall-cmd --permanent --add-port=139/tcp --add-port=445/tcp

1）Port 137 (UDP) - NetBIOS 名字服务 ； nmbd

2）Port 138 (UDP) - NetBIOS 数据报服务

3）Port 139 (TCP) - 文件和打印共享 ； smbd （基于SMB(Server Message Block)协议，主要在局域网中使用，文件共享协议）

4）Port 389 (TCP) - 用于 LDAP (Active Directory Mode)

5）Port 445 (TCP) - NetBIOS服务在windos 2000及以后版本使用此端口, (Common
Internet File System，CIFS，它是SMB协议扩展到Internet后，实现Internet文件共享)

6）Port 901 (TCP) - 用于 SWAT，用于网页管理Samba

systemctl restart firewalld.service //然后重启防火墙服务，一次不行，再执行一次。。。。

六 Android端的使用

在Linux上用samba搭建一个共享服务器(Windows的文件共享也可以)，把手机和电脑连上相同的局域网，然后Android上用ES文件浏览器就可以直接查看电脑的文件。连上局域网就可以了！哟嚯霍！windows上面更简单。。。百度一下吧。

一个小意外：

在使用了一个下午后，firewalld 的zone 不知道为什么变成了 另一个很长的名字。。
在重新在 /etc/firewalld/firewalld.conf 把zone 改为了public ，ok 局域网又成功连接
